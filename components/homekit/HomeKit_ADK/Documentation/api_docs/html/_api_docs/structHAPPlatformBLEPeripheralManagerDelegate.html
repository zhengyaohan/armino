struct HAPPlatformBLEPeripheralManagerDelegate¶

Delegate that is used to monitor read, write, and subscription requests from remote central devices.

Public Members

void *context¶

Client context pointer. Will be passed to callbacks.

void (*handleConnectedCentral)(HAPPlatformBLEPeripheralManagerRef blePeripheralManager, HAPPlatformBLEPeripheralManagerConnectionHandle connectionHandle, void *context)¶

Invoked when a connection has been established in response to the advertising data that has been set through HAPPlatformBLEPeripheralManagerStartAdvertising.

  • If a connection is established through other means, it is not considered a HomeKit connection and must not lead to the invocation of this callback.

Parameters

  • blePeripheralManager: BLE peripheral manager.

  • connectionHandle: Connection handle of the connected central.

  • context: The context pointer of the BLE peripheral manager delegate structure.

void (*handleDisconnectedCentral)(HAPPlatformBLEPeripheralManagerRef blePeripheralManager, HAPPlatformBLEPeripheralManagerConnectionHandle connectionHandle, void *context)¶

Invoked when a connection that was reported to the handleConnectedCentral callback has been terminated.

Parameters

  • blePeripheralManager: BLE peripheral manager.

  • connectionHandle: Connection handle of the disconnected central.

  • context: The context pointer of the BLE peripheral manager delegate structure.

HAPError (*handleReadRequest)(HAPPlatformBLEPeripheralManagerRef blePeripheralManager, HAPPlatformBLEPeripheralManagerConnectionHandle connectionHandle, HAPPlatformBLEPeripheralManagerAttributeHandle attributeHandle, void *bytes, size_t maxBytes, size_t *numBytes, void *context)¶

Invoked when a read request is received on an attribute that has been registered through HAPPlatformBLEPeripheralManagerAddCharacteristic or HAPPlatformBLEPeripheralManagerAddDescriptor.

  • The supplied buffer should have space for kHAPPlatformBLEPeripheralManager_MaxAttributeBytes bytes. It is left to the BLE peripheral manager implementation to then transfer the full buffer over a sequence of central-initiated “Read Request” and “Read Blob Request” operations to the central. This callback should only be invoked again once the full data has been transmitted.

Return

kHAPError_None If successful.

Return

kHAPError_InvalidState If a read on that characteristic is not allowed in the current state.

Return

kHAPError_OutOfResources If the supplied buffer is not large enough.

Parameters

  • blePeripheralManager: BLE peripheral manager.

  • connectionHandle: Connection handle of the central that sent the request.

  • attributeHandle: Attribute handle that is being read.

  • [out] bytes: Buffer to fill read response into.

  • maxBytes: Capacity of buffer.

  • [out] numBytes: Length of data that was filled into buffer.

  • context: The context pointer of the BLE peripheral manager delegate structure.

HAPError (*handleWriteRequest)(HAPPlatformBLEPeripheralManagerRef blePeripheralManager, HAPPlatformBLEPeripheralManagerConnectionHandle connectionHandle, HAPPlatformBLEPeripheralManagerAttributeHandle attributeHandle, void *bytes, size_t numBytes, void *context)¶

Invoked when a write request is received on an attribute that has been registered through HAPPlatformBLEPeripheralManagerAddCharacteristic or HAPPlatformBLEPeripheralManagerAddDescriptor.

  • The supplied buffer must support writes up to kHAPPlatformBLEPeripheralManager_MaxAttributeBytes bytes. It is left to the BLE peripheral manager implementation to assemble fragments of potential “Prepare Write Request” and “Execute Write Request” operations. This callback should only be invoked once the full data has been received.

Return

kHAPError_None If successful.

Return

kHAPError_InvalidState If a write on that characteristic is not allowed in the current state.

Return

kHAPError_InvalidData If the request data has an invalid format.

Return

kHAPError_OutOfResources If there are not enough resources to handle the request.

Parameters

  • blePeripheralManager: BLE peripheral manager.

  • connectionHandle: Connection handle of the central that sent the request.

  • attributeHandle: Attribute handle that is being read.

  • bytes: Buffer that contains the request data.

  • numBytes: Length of data in buffer.

  • context: The context pointer of the BLE peripheral manager delegate structure.

void (*handleReadyToUpdateSubscribers)(HAPPlatformBLEPeripheralManagerRef blePeripheralManager, HAPPlatformBLEPeripheralManagerConnectionHandle connectionHandle, void *context)¶

Invoked when the BLE peripheral manager is again ready to send characteristic value updates through HAPPlatformBLEPeripheralManagerSendHandleValueIndication.

Parameters

  • blePeripheralManager: BLE peripheral manager.

  • connectionHandle: Connection handle of the central that is ready to receive events.

  • context: The context pointer of the BLE peripheral manager delegate structure.
