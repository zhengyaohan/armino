Detailed Description¶

Include platform-specific feature list of features to enable or disable.

Each platform can enable or disable features as required to create a smaller PALs with only the necessary components for a product.

The following features can be toggled on/off:

  • HAP_CAMERA_AAC_ELD - Enables AAC-ELD Audio codec related platform APIs for IP cameras.

  • HAP_DYNAMIC_MEMORY_ALLOCATION - This feature flag may be used to dynamically allocate the inbound and outbound buffers as well as event notification objects based upon session activity to minimize the idle memory usage for IP based accessories. This feature should only be enabled if there are concerns regarding idle memory usage. NOTE: the peak RAM usage is dependent upon the number of controllers simultaneously exchanging data with the accessory.

  • HAP_FEATURE_ACCESS_CODE - Enables HAP Access Code Service related platform APIs. This feature allows capability to use PIN codes on locks.

  • HAP_FEATURE_ACCESSORY_METRICS - Enables accessory metrics related PAL APIs. For more information, please refer to Accessory Metrics Service.

  • HAP_FEATURE_BATTERY_POWERED_RECORDER - Enables the support for battery operated IP cameras.

  • HAP_FEATURE_CAMERA - Enables camera and camera recorder related platform APIs.

  • HAP_FEATURE_DIAGNOSTICS_SERVICE - Enables Diagnostics Service related platform APIs.

  • HAP_FEATURE_FILE_STORAGE - Enables file management related platform APIs for accessory platforms that support a writable file system.

  • HAP_FEATURE_HDS_TRANSPORT_OVER_HAP - Enables HomeKit Data Stream to operate over HAP, which allows for UARP or Diagnostics to work over BLE and Thread.

  • HAP_FEATURE_IP - Enables IP support.

  • HAP_FEATURE_MFI_HW_AUTH - Enables Apple Authentication Coprocessor related platform APIs for accessory authentication. For more information, please refer to Accessory Authentication.

  • HAP_FEATURE_MFI_TOKEN_AUTH - Enables MFi software token related platform APIs for accessory authentication. For more information, please refer to Accessory Authentication.

  • HAP_FEATURE_MICROPHONE - Enables microphone related platform APIs. This is required by applications such as Remote with Siri support and IP cameras.

  • HAP_FEATURE_NFC - Enables NFC related platform APIs. This needs to be enabled if an accessory platform supports NFC.

  • HAP_FEATURE_NFC_ACCESS - Enables NFC Access Service related platform APIs. This features allows an accessory to implement tap to unlock.

  • HAP_FEATURE_PLATFORM_BLE - This feature flag indicates whether the platform supports BLE. HAP over BLE also requires application to enable BLE explicitly such as configuring the BLE transport in the accessory server.

  • HAP_FEATURE_SPEAKER - Enables speaker related platform APIs. This is required by applications such as IP cameras.

  • HAP_FEATURE_THREAD - Enables Thread capabilities for platforms that support Thread radio.

  • HAP_FEATURE_THREAD_CERTIFICATION_OVERRIDES - This feature flag disables the Thread viability check used to ensure that a viable Thread network is reachable and fallback on BLE if a Thread network is not reachable. This feature flag can be turned on to disable this behavior for the purpose of Open Thread certification.

  • HAP_FEATURE_UARP_SUPPORT - Enables Unified Accessory Restore Protocol related platform APIs. For more information, please refer to Firmware Update Service.

  • HAP_FEATURE_WAC - Enables Wireless Accessory Configuration support for IP accessories. Please refer to Section 3.3 of HomeKit Accessory Protocol Specification available on the MFi portal for more details.

  • HAP_FEATURE_WIFI_RECONFIGURATION - Enables the support for Wifi Reconfiguration for IP accessories.

  • HAP_FEATURE_WIFI_ROUTER - Enables wifi router capabilities.
